Use Conventional Commits style for all commit messages (feat:, fix:, chore:, refactor:, test:, docs:).

**Why:** User explicitly requested this as a standard across projects. Added to global CLAUDE.md.

**How to apply:** Every git commit message must start with a type prefix. Use the most specific type that fits (e.g., `feat:` for new features, `fix:` for bugs, `chore:` for deps/config).
